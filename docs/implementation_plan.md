# 부동산 탭 버그 수정 및 UI/UX 구조 개편 계획

## 1. 부동산 탭 미출력 버그 수정
- **문제 원인**: 네비게이션 탭 전환 시 화면을 제어하는 `switchView` 함수의 `views` 목록에 `realestate-view`가 등록되지 않아, 화면 숨김(`hidden`) 클래스가 해제되지 않는 현상.
- **해결 방안**: `index.html`의 `views` 객체에 `'realestate-view'` 속성을 추가하여 정상적으로 화면이 전환되도록 수정합니다.

## 2. 투자자산 상세 페이지 신설
- **문제 인식**: 포트폴리오 탭에서 투자 자산(주식, ETF 등)의 상세 내역(주식 수, 단가, 현재가 등)을 한눈에 보기에는 공간이 협소함.
- **해결 방안**: 
  - `index.html` 내에 새로운 별도 뷰(View)인 `<div id="invest-detail-view">`를 신설합니다.
  - 포트폴리오 탭의 '투자 자산' 카드 상단에 **[상세보기]** 버튼을 추가하고, 이 버튼을 누르면 신설된 '투자자산 상세 페이지'로 화면이 전환되도록 구성합니다.
  - 상세 페이지 내에는 포트폴리오 데이터 중 '투자 자산' 그룹의 항목들만 모아 시각적인 카드 형태로 크게 보여주며, 총 주식수 및 1주당 단가, 총 금액을 큼직하게 렌더링합니다. (뒤로가기 버튼 포함)

## 3. 대시보드 월 수입/지출 카드 병합
- **기존 구조**: 대시보드 상단에 1) 총 자산, 2) 이번 달 수입, 3) 이번 달 지출 카드가 3개로 나뉘어 출력됨 (md:grid-cols-3).
- **개선 구조**: 
  - '이번 달 수입'과 '이번 달 지출' 카드를 하나의 카드(Card) 컴포넌트로 병합합니다.
  - 상단 그리드 레이아웃을 2열(md:grid-cols-2)로 변경하여 1열엔 '총 자산', 2열엔 '수입/지출 내역'을 나란히 배치해 시인성을 높입니다.
  - 하나의 카드 안에서 수입은 파란색 화살표와 텍스트로, 지출은 빨간색 화살표와 텍스트로 좌우 또는 위아래로 깔끔하게 대비시켜 표시합니다.

> 투자자산 상세 페이지 진입을 별도의 하단 탭(Tab)이 아닌, **포트폴리오 화면 내의 [상세보기] 버튼을 통해 진입**하는 형태(Sub-page 방식)로 기획했습니다. 하단 탭 공간이 이미 5개(대시보드, 현금흐름, 자산추이, 포트폴리오, 부동산)로 꽉 차 있기 때문입니다. 이 진입 방식이 괜찮으신지 확인 부탁드립니다!
