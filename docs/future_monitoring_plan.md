# 🚀 NetVisualizer V2: 향후 개선 및 모니터링 계획안

지금까지 구축한 `NetVisualizer`는 단순히 숫자를 기록하는 가계부를 넘어, 주식/ETF의 실시간 가격을 반영하고 부동산 청약 자금까지 시뮬레이션할 수 있는 훌륭한 **개인 종합 자산 관리 대시보드**가 되었습니다.

앞으로 앱을 더욱 강력하고 유용하게 발전시키기 위해, **무엇을 모니터링하고 어떤 방향으로 개선해 나갈지**에 대한 아이디어를 제안합니다.

---

## 1. 📊 시스템 성능 및 안정성 모니터링 (앱 최적화)

사용하면서 데이터가 쌓일수록 앱이 느려지거나 버그가 생기지 않는지 체크하는 항목입니다.

- **API 응답 속도 체크**: 구글 시트 데이터가 1년, 2년 치 쌓였을 때 앱을 처음 켤 때 로딩되는 시간(GET 요청 속도)을 모니터링합니다. 너무 느려진다면 과거 데이터는 별도 시트로 분리하는 아카이빙(Archiving) 기능을 고려합니다.
- **주가 파싱 API 안정성**: 네이버 금융 폴링 API(`GET_NAVER_PRICE`)가 에러 없이 잘 작동하는지 한 달 주기로 점검합니다. 만약 차단되거나 구조가 바뀐다면 Yahoo Finance 등 대체 API 플랜을 준비합니다.

## 2. 💰 개인 재무 목표 모니터링 (사용자 관점)

앱이 사용자(회원님)에게 어떤 인사이트를 줄 수 있을지 고민하는 영역입니다.

- **월별 잉여 자금(저축률) 추적**: 매달 들어오는 수입 대비, 낭비되는 지출을 제외하고 '실제로 투자나 안전자산으로 넘어간 비율(저축률)'이 우상향하고 있는지 대시보드에 지표화해 봅니다.
- **부동산 청약 타임라인 모니터링**: 3기 신도시의 실제 분양 공고일과 내 자산의 '총 가용 자금(대출 포함)'의 성장 곡선을 교차 분석하여, 특정 시점에 내가 청약을 넣을 수 있을지 '청약 D-Day 분석' 기능을 추가해 볼 수 있습니다.

## 3. 🛠️ 향후 추가해 볼 만한 혁신 기능들 (Next Level)

앱을 진짜 상용 서비스급으로 끌어올릴 수 있는 아이디어입니다.

- **알림(Push Notification) 기능**: PWA(웹앱)의 장점을 살려, 매월 1일에 "이번 달 가계부를 마감하세요!" 혹은 "특정 주식/ETF가 목표가에 도달했습니다" 같은 푸시 알림 기능을 도입해 봅니다.
- **자동화된 자산 연동 (오픈뱅킹)**: 현재는 구글 시트를 통해 수동/반자동으로 관리하지만, 향후 Python 등을 이용해 카드사 결제 내역이나 은행 잔고를 구글 시트로 자동 스크래핑하여 앱에 알아서 뜨게 만드는 파이프라인을 구축해 볼 수 있습니다.
- **테마 커스터마이징**: 현재의 밝은 테마 외에도, 주식 앱들처럼 멋진 다크 모드(Dark Mode)를 추가하여 밤에도 눈 아프지 않게 자산을 확인할 수 있게 만듭니다.
