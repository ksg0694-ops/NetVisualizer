# 스마트 가계부 V2 (NetVisualizer_v01)

## 📌 개요
개인 가계부 및 자산 포트폴리오, 부동산 청약 관리를 위한 반응형 스마트 대시보드 웹앱입니다. 
구글 스프레드시트를 백엔드 데이터베이스(DB)로 사용하며, 프론트엔드는 HTML/Vanilla JS 기반의 PWA(Progressive Web App)로 동작하여 모바일 앱처럼 설치 및 활용이 가능합니다.

## 🏗️ 시스템 아키텍처 및 파일 구성

### 1. 프론트엔드 (Frontend)
- **`index.html`**
  - 앱의 전체 UI와 비즈니스 로직(데이터 파싱, 차트 렌더링, API Fetch)을 담당하는 단일 페이지 어플리케이션(SPA) 역할을 합니다.
  - TailwindCSS를 활용한 반응형 UI와 Chart.js를 이용한 데이터 시각화, Leaflet.js를 이용한 지도 시각화를 제공합니다.
  - 주요 뷰(View): 대시보드, 포트폴리오, 투자 자산 상세, 현금 흐름, 장기 자산 추이, 부동산 청약 지도
- **`manifest.json` & `sw.js`**
  - PWA 구동을 위한 메타데이터 및 서비스 워커 설정 파일입니다.
  - 앱 설치 기능을 제공하고, 오프라인 상태이거나 네트워크가 불안정할 경우 캐싱된 리소스와 데모 데이터를 보여주도록 설정되어 있습니다.

### 2. 백엔드 (Backend - Google Apps Script)
- **`code.gs`**
  - 구글 스프레드시트에 연동되어 배포된 백엔드 API 엔진입니다.
  - **`doGet(e)`**: 스프레드시트("수입지출 내역", "자산추이내역", "포트폴리오") 데이터를 읽어 프론트엔드에 JSON 객체로 반환합니다.
  - **`doPost(e)`**: 프론트엔드에서 수정된 데이터를 스프레드시트에 안전하게 기록합니다. 데이터 꼬임을 방지하기 위해 `LockService`가 적용되어 있습니다.
  - **`GET_NAVER_PRICE(code)`**: 네이버 금융 API(`polling.finance.naver.com`)를 직접 호출하여, 국내 주식/ETF 등의 실시간 가격 데이터를 가져오는 구글 시트용 커스텀 함수입니다.

## 🔄 주요 프로세스 및 최신 기능

### 1. 데이터 불러오기 및 가격 동기화
1. 사용자가 앱에 접속하면 `index.html`의 초기화 로직이 Apps Script Web App URL로 **GET 요청**을 보냅니다.
2. 구글 시트 내부적으로 `GET_NAVER_PRICE` 함수가 주가 정보를 최신화하며, 연동된 포트폴리오 자산 데이터를 JSON으로 반환합니다.

### 2. 부동산 청약 및 대출 시뮬레이션 (신규)
1. **부동산 청약 탭**에서 주요 3기 신도시(하남 교산, 남양주 왕숙 등)의 예상 분양가와 위치를 **인터랙티브 지도**로 보여줍니다.
2. 현재 사용자의 보유 현금 및 안전 자산을 기준으로 **목표 금액 달성률**을 렌더링합니다.
3. 현재 자산의 50%를 예상 대출 가능 금액으로 산정하여 **총 가용 자금** 시뮬레이션을 제공합니다.

### 3. 투자 자산 상세 보기 (신규)
- 포트폴리오 탭의 **투자 자산** 항목에 대해 개별 주식/ETF의 **보유 수량**, **1주당 단가**, **평가 금액**을 모아볼 수 있는 독립적인 상세 뷰(View)를 지원합니다.

### 4. 거래 내역 추가 및 포트폴리오 갱신
- 우측 하단의 플로팅 버튼(+)을 통해 수입/지출 내역을 작성하여 시트에 안전하게 기록(`doPost`)합니다.
- 포트폴리오 탭의 폼에서 자산의 금액을 수정하면 전체 시트가 실시간 갱신됩니다.

### 5. 시각화 및 간이 AI 분석 기능
- **차트**: 도넛 차트(지출/자산 비중), 막대/선 그래프(순이익 추세).
- **AI 코멘트**: 과거 3개월 평균과 이번 달을 비교하여 과도한 지출에 대한 경고(Warning) 메시지 제공.

## 🚀 개발 및 테스트 가이드
1. **프론트엔드 테스트**
   - 프로젝트 폴더에서 로컬 서버를 실행하여 접속합니다. (예: `python -m http.server 8000`)
2. **백엔드(Apps Script) 수정 시 주의사항**
   - `code.gs` 코드를 수정한 뒤에는 스프레드시트 Apps Script 편집기에서 반드시 **[배포] -> [새 버전 만들기]**를 통해 재배포해야 프론트엔드 앱 및 네이버 주가 파싱 기능이 반영됩니다.
