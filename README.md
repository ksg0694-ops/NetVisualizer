# 스마트 가계부 V2 (NetVisualizer_v01)

## 📌 개요
개인 가계부 및 자산 포트폴리오 관리를 위한 반응형 스마트 대시보드 웹앱입니다. 
구글 스프레드시트를 백엔드 데이터베이스(DB)로 사용하며, 프론트엔드는 HTML/Vanilla JS 기반의 PWA(Progressive Web App)로 동작하여 모바일 앱처럼 설치 및 활용이 가능합니다.

## 🏗️ 시스템 아키텍처 및 파일 구성

### 1. 프론트엔드 (Frontend)
- **`index.html`**
  - 앱의 전체 UI와 비즈니스 로직(데이터 파싱, 차트 렌더링, API Fetch)을 담당하는 단일 페이지 어플리케이션(SPA) 역할을 합니다.
  - TailwindCSS를 활용한 반응형 UI와 Chart.js를 이용한 데이터 시각화를 제공합니다.
  - 주요 뷰(View): 대시보드, 포트폴리오, 현금 흐름, 장기 자산 추이
- **`manifest.json` & `sw.js`**
  - PWA 구동을 위한 메타데이터 및 서비스 워커 설정 파일입니다.
  - 앱 설치 기능을 제공하고, 오프라인 상태이거나 네트워크가 불안정할 경우 캐싱된 리소스와 데모 데이터를 보여주도록 설정되어 있습니다.

### 2. 백엔드 (Backend - Google Apps Script)
- **`code.gs`**
  - 구글 스프레드시트에 연동되어 배포된 백엔드 API 엔진입니다.
  - **`doGet(e)`**: 스프레드시트("수입지출 내역", "자산추이내역", "포트폴리오") 데이터를 읽어 프론트엔드에 JSON 객체로 반환합니다.
  - **`doPost(e)`**: 프론트엔드에서 수정된 데이터를 스프레드시트에 안전하게 기록합니다. 데이터 꼬임을 방지하기 위해 `LockService`가 적용되어 있습니다.

## 🔄 주요 프로세스 및 데이터 흐름

### 1. 데이터 불러오기 (Read)
1. 사용자가 앱에 접속하면 `index.html`의 초기화 로직이 Apps Script Web App URL로 **GET 요청**을 보냅니다.
2. `code.gs`는 시트의 데이터를 읽어 JSON 문자열로 반환하고, 프론트엔드에서는 이를 파싱하여 화면 및 차트에 렌더링합니다.

### 2. 거래 내역 추가 (Write - `addTransaction`)
1. 우측 하단의 플로팅 버튼(+)을 통해 수입/지출 내역을 작성합니다.
2. 저장 버튼 클릭 시, 작성된 데이터를 JSON으로 감싸 Web App URL에 **POST 요청**을 전송합니다 (`action: "addTransaction"`).
3. `code.gs`는 요청을 받아 "수입지출 내역" 시트의 맨 아래에 새로운 행(Row)으로 데이터를 추가합니다.

### 3. 포트폴리오 갱신 (Write - `updatePortfolio`)
1. '포트폴리오' 탭에서 [금액 수정] 버튼을 눌러 각 자산의 현재 가치를 갱신합니다.
2. 수정한 데이터를 2차원 배열 형태로 묶어 **POST 요청**을 전송합니다 (`action: "updatePortfolio"`).
3. `code.gs`는 기존 "포트폴리오" 시트의 내용을 비우고(clear), 새로운 데이터로 전체를 덮어씁니다.

### 4. 시각화 및 간이 AI 분석 기능
- **차트**: 불러온 데이터를 바탕으로 도넛 차트(지출 비중, 자산 비중), 막대/선 그래프(순이익 추세), 장기 자산 달성률 등을 시각화합니다.
- **AI 코멘트**: 과거 3개월 간의 평균 지출액과 이번 달의 카테고리별 지출액을 비교하여, 평소보다 과도하게 지출된 카테고리를 잡아내 경고(Warning)해 주는 자체 규칙 기반 분석을 제공합니다.

## 🚀 개발 및 테스트 가이드
1. **프론트엔드 테스트**
   - 프로젝트 폴더에서 로컬 서버를 실행하여 접속합니다. (예: `python -m http.server 8000`)
   - 데스크탑 및 모바일 브라우저를 통해 UI/UX를 테스트합니다.
2. **백엔드(Apps Script) 수정 시 주의사항**
   - `code.gs` 코드를 수정한 뒤에는 스프레드시트 Apps Script 편집기에서 반드시 **[배포] -> [새 버전 만들기]**를 통해 재배포해야 프론트엔드 앱에 변경사항이 반영됩니다.
